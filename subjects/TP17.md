# TP17 — Normal Mapping

## Objectif

Jusqu'ici les matériaux utilisent la **normale géométrique** pour calculer les rebonds de lumière.
Ce TP introduit le **normal mapping** : perturber la normale de shading en lisant une texture,
simulant un relief visuel *sans ajouter de géométrie*.

Ce TP introduit :
- `Vec3d tangent` dans `HitRecord` — l'espace tangent à la surface
- Calcul du **TBN** pour `Sphere` et `Triangle` (Tangent / Bitangent / Normal)
- `struct Mat3` — matrice 3×3, multiplication par un vecteur
- `BumpTexture` — normale procédurale par gradient de Perlin
- `normal_mapped()` — décorateur de matériau (pattern décorateur avec lambdas)

---

## 1. Le problème : une sphère qui *semble* rugueuse

Deux façons d'obtenir une surface rugueuse :

| Approche | Coût | Résultat |
|---|---|---|
| Géométrie fine (millions de polygones) | Très coûteux (mémoire + BVH) | Ombres correctes |
| **Normal mapping** | Quasi-nul | Illusion convaincante |

La normale de shading remplace la normale géométrique *uniquement pour les rebonds de lumière*.
Les ombres propres (shadow rays) utilisent encore la géométrie réelle — c'est la limite du normal mapping.

---

## 2. L'espace tangent

Pour interpréter une normal map, on définit un **repère local** sur la surface :

```
        N  (normale géométrique, vers l'extérieur)
        │
        │   B  (bitangente = N × T)
        │  ╱
        │ ╱
        └────── T  (tangente, direction croissante de U)
```

Ce repère `{T, B, N}` est la **matrice TBN**. Une normal map stocke des normales dans cet
espace local :
- `(0, 0, 1)` = normale plate (= normale géométrique, aucun bump)
- `(1, 0, 0)` = surface inclinée dans la direction T
- Les valeurs intermédiaires = toutes les inclinaisons possibles

Pour revenir en espace monde :

```
normal_monde = T * ts.x  +  B * ts.y  +  N * ts.z
```

## 3. struct Mat3

Crée `src/core/mat3.h` :

```cpp
#pragma once
#include "core/vec3.h"

namespace rt {
    struct Mat3 {
        Vec3d col0, col1, col2;  // colonnes : T, B, N pour TBN

        Vec3d operator*(const Vec3d& v) const {
            return col0 * v.x + col1 * v.y + col2 * v.z;
        }
    };
}
```

Usage :

```cpp
Mat3 tbn{ tangent, bitangent, normal };
Vec3d world_normal = tbn * tangent_space_normal;
```

## 4. Tangent sur Sphere

La tangente se déduit de phi (angle horizontal de l'UV mapping sphérique) :

```
phi     = atan2(-outward_normal.z, outward_normal.x) + pi
tangent = Vec3d(-sin(phi), 0.0, cos(phi))
```

À stocker dans le `HitRecord` retourné par `hit()`.

**Cas dégénéré aux pôles** (n ≈ (0,±1,0) : phi = atan2(0,0) = indéfini) :

```cpp
if (tangent.length() < 1e-6)
    tangent = Vec3d(1, 0, 0);  // valeur arbitraire valide
```

**Vérification** : `dot(tangent, normal) ≈ 0` et `|tangent| = 1`.

## 5. Tangent sur Triangle

Pour un triangle avec des UV par sommet, la tangente se calcule à partir des **différentielles UV** :

```
dPos1 = B - A                     dUV1 = (uv_B.x - uv_A.x,  uv_B.y - uv_A.y)
dPos2 = C - A                     dUV2 = (uv_C.x - uv_A.x,  uv_C.y - uv_A.y)

det = dUV1.x * dUV2.y  -  dUV2.x * dUV1.y
T   = (dUV2.y * dPos1 - dUV1.y * dPos2) / det
```

Si `|det| < 1e-8` (UV dégénérés) : utilise `(B - A).normalized()`.

**Orthogonalisation de Gram-Schmidt** (T doit être ⊥ à la normale) :

```
T = (T - dot(T, normal) * normal).normalized()
```

Précompute T dans le constructeur de `Triangle` (elle ne change pas). Stocke dans le `HitRecord`.

## 7. BumpTexture — normal map procédurale

Ajoute dans `src/materials/texture.h` une texture qui génère une normale tangente via le
**gradient numérique du bruit de Perlin** :

```cpp
class BumpTexture : public Texture {
    PerlinNoise noise;
    double scale, strength;
public:
    BumpTexture(double scale = 4.0, double strength = 1.0);

    // Retourne une normale tangente, composantes dans [-1, 1]
    Vec3d sample(double u, double v, const Vec3d& p) const override;
};
```

> **Convention** : contrairement aux textures de couleur (qui retournent [0, 255]),
> `BumpTexture` retourne directement des composantes dans `[-1, 1]`.
> Le décorateur `normal_mapped()` connaît cette convention.

Algorithme de `sample()` — gradient par différences finies :

```
eps  = 0.01
h    = noise.noise(p * scale)
dhu  = (noise.noise((p + Vec3d(eps, 0, 0)) * scale) - h) / eps
dhv  = (noise.noise((p + Vec3d(0, eps, 0)) * scale) - h) / eps

return Vec3d(-dhu * strength, -dhv * strength, 1.0).normalized()
```

Interprétation : `-dh/du` et `-dh/dv` donnent la pente de la surface bumpée. La normale
est perpendiculaire à cette surface. À `strength = 0`, le résultat est exactement `(0, 0, 1)`.

## 8. normal_mapped() — décorateur de matériau

Dans `src/materials/materials.h` (déclaration) et `src/materials/materials.cpp` (implémentation) :

```cpp
// Déclaration dans materials.h :
Material normal_mapped(Material base, TexturePtr normal_map);
```

Implémentation :

```cpp
Material normal_mapped(Material base, TexturePtr nmap) {
    return [base, nmap](const Ray& r, const HitRecord& rec) -> std::optional<Scatter> {
        Vec3d ts = nmap->sample(rec.u, rec.v, rec.point).normalized();

        Vec3d T = rec.tangent;
        Vec3d N = rec.normal;
        Vec3d B = N.cross(T);
        Mat3  tbn{T, B, N};

        HitRecord perturbed  = rec;
        perturbed.normal     = (tbn * ts).normalized();

        return base(r, perturbed);
    };
}
```

**Pattern décorateur** : `normal_mapped` *enveloppe* n'importe quel `Material` — lambertien,
métal, verre. Il modifie uniquement la normale avant de déléguer.
