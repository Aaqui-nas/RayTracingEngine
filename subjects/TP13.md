# TP13 — Maillages triangulaires et chargement OBJ

## Introduction

Jusqu'ici, notre ray tracer ne sait dessiner que des sphères et des plans. Pour représenter n'importe quel objet 3D — un lapin, une théière, un casque — il nous faut une primitive plus générale : **le triangle**.

Un maillage (*mesh*) 3D n'est rien d'autre qu'un ensemble de triangles. Le format OBJ (Wavefront) est le format d'échange le plus courant pour ces maillages : simple, lisible à l'œil nu, exportable depuis Blender, Maya, ou n'importe quel modeleur 3D.

À la fin de ce TP, vous pourrez importer n'importe quel modèle OBJ dans votre ray tracer.

**Point de départ** : le code issu de TP12bis (namespaces, CMake, Camera, materials.h/.cpp).

## Section 1 — Le triangle : la brique de base

### Représentation

Un triangle est défini par **trois sommets** dans l'espace : A, B, C.

```
        A
       / \
      /   \
     /     \
    B-------C
```

### La normale

La **normale** d'un triangle est le vecteur perpendiculaire à sa surface. On la calcule par le **produit vectoriel** des deux arêtes issues de A :

```
E1 = B - A
E2 = C - A
normale = (E1 × E2).normalized()
```

La direction de la normale dépend de l'**ordre des sommets**. Dans notre cas, on garde la normale telle quelle et on détermine si le rayon frappe la face avant ou arrière au moment de l'intersection, comme on le fait pour la sphère.

### Interface de la classe Triangle

Crée `triangle.h` avec cette structure :

```cpp
#pragma once
#include "shape.h"

namespace rt {

class Triangle : public Shape {
public:
    Vec3d A, B, C;
    Vec3d normal;

    Triangle(Vec3d a, Vec3d b, Vec3d c, Material mat)
        : Shape(mat), A(a), B(b), C(c)
    {
        normal = (B - A).cross(C - A).normalized();
    }

    std::optional<HitRecord> hit(const Ray& ray, double tmin, double tmax) const override;
};

} // namespace rt
```

Triangle hérite de Shape, donc il s'intègre directement dans la scène, exactement comme Sphere ou Plane.

## Section 2 — L'algorithme de Möller-Trumbore

La question centrale : **est-ce que ce rayon touche ce triangle, et si oui, où ?**

### Idée générale

Un point P sur le triangle peut s'écrire en **coordonnées barycentriques** :

```
P = A + u*(B-A) + v*(C-A)
```

avec u ≥ 0, v ≥ 0 et u+v ≤ 1.

Un point sur le rayon s'écrit :

```
P = origin + t * direction
```

On cherche t, u, v tels que ces deux expressions soient égales.

### Les étapes détaillées

Voici l'algorithme complet, à traduire directement en C++ :

```
// Données d'entrée
Ray     : origin O, direction D
Triangle: sommets A, B, C

// Étape 1 : arêtes du triangle
E1 = B - A
E2 = C - A

// Étape 2 : vecteur h (perpendiculaire à D et E2)
h = cross(D, E2)

// Étape 3 : déterminant
det = dot(E1, h)
Si |det| < 1e-8 → parallèle, retourner nullopt

f = 1.0 / det

// Étape 4 : première coordonnée barycentrique u
s = O - A
u = f * dot(s, h)
Si u < 0 ou u > 1 → retourner nullopt

// Étape 5 : deuxième coordonnée barycentrique v
q = cross(s, E1)
v = f * dot(D, q)
Si v < 0 ou (u + v) > 1 → retourner nullopt

// Étape 6 : distance t
t = f * dot(E2, q)
Si t < tmin ou t > tmax → retourner nullopt

// Intersection trouvée !
point = O + t * D
```

### Construction du HitRecord

Une fois t et le point d'intersection trouvés, on construit le `HitRecord` de la même façon que pour Sphere :

```cpp
HitRecord rec;
rec.t = t;
rec.point = ray.at(t);
// Détermine si le rayon frappe la face avant ou arrière
bool front = dot(ray.direction, normal) < 0;
rec.normal = front ? normal : -normal;
rec.front_face = front;
rec.material = material;
return rec;
```

## Section 3 — Le format OBJ

Le format Wavefront OBJ est un fichier texte. Voici un exemple minimal, un tétraèdre :

```obj
# Commentaire : tétraèdre régulier
# 4 sommets, 4 faces triangulaires

v  0.0    1.0    0.0
v  0.0   -0.5    1.0
v  0.866 -0.5   -0.5
v -0.866 -0.5   -0.5

f 1 2 3
f 1 3 4
f 1 4 2
f 2 4 3
```

Les lignes importantes :

| Préfixe | Signification                        | Exemple            |
|---------|--------------------------------------|--------------------|
| `v`     | Sommet (vertex) : x y z              | `v 1.0 0.5 -2.0`  |
| `f`     | Face : indices des sommets (1-based) | `f 1 3 4`          |
| `#`     | Commentaire, ignoré                  |                    |
| `vn`    | Normale (ignoré dans ce TP)          |                    |
| `vt`    | Coordonnée UV (ignoré dans ce TP)    |                    |

**Attention** : les indices dans `f` commencent à **1**, pas à 0. Il faut donc faire `vertices[i-1]`.

Les lignes `f` peuvent aussi contenir des formats étendus comme `f 1/2/3 4/5/6 7/8/9` (vertex/texture/normale). Pour extraire l'entier avant le premier `/`, utilise `std::stoi(token.substr(0, token.find('/')))`.

## Section 4 — Mesh : une collection de triangles

Un `Mesh` est simplement un `Shape` qui contient un `std::vector<Triangle>`.

Interface à créer dans `mesh.h` :

```cpp
#pragma once
#include "triangle.h"
#include <vector>

namespace rt {

class Mesh : public Shape {
public:
    std::vector<Triangle> triangles;

    Mesh() : Shape(Material{}) {}

    void add(Triangle t) { triangles.push_back(t); }

    std::optional<HitRecord> hit(const Ray& ray, double tmin, double tmax) const override;
};

} // namespace rt
```

Son `hit()` fonctionne exactement comme `Scene::hit()` : il teste chaque triangle un par un et retourne le plus proche (sans l'indice, juste le HitRecord).

## Note sur les performances

Sans structure d'accélération, chaque rayon teste **tous** les triangles du mesh un par un. Un modèle avec 10 000 triangles fera 10 000 tests par rayon. Pour une image 400×225 avec 64 samples et 8 rebonds :

```
400 × 225 × 64 × 8 × 10 000 = 46 milliards de tests
```

C'est très lent. Le **TP14** introduira un BVH (Bounding Volume Hierarchy) qui réduit la complexité de O(n) à O(log n) par rayon — permettant d'utiliser des modèles avec des millions de triangles.

Pour l'instant, utilisez des modèles légers (< 1000 triangles) ou lancez des rendus basse résolution :
```bash
./raytracer 0.5 16   # 200×112, 16 samples — rapide
```
