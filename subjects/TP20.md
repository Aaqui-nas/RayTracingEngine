# TP20 — Matériaux PBR : BRDF Cook-Torrance

## Objectif

Jusqu'ici, les matériaux du renderer sont soit parfaitement diffus (Lambertien), soit parfaitement
métalliques (miroir), soit en verre. Aucun matériau réel ne se comporte ainsi. Ce TP introduit le
modèle **PBR (Physically Based Rendering)** à deux paramètres — **roughness** et **metalness** —
qui reproduit l'apparence de n'importe quel matériau opaque : or poli, cuivre rouillé, plastique
lustré, béton mat.

Ce TP introduit :
- Les **closures multi-paramètres** : une factory capturant 4 paramètres physiques (albedo,
  roughness, metalness, IOR)
- Le **namespace `physics`** : isoler des fonctions mathématiques pures dans un sous-namespace
- La **BRDF Cook-Torrance** : trois composantes D, F, G modélisant les microfacettes
- L'**importance sampling GGX** (extension) : réduire la variance sur les matériaux brillants

---

## 1. Qu'est-ce qu'une BRDF ?

La **Bidirectional Reflectance Distribution Function** (BRDF) `f_r(ω_i, ω_o)` décrit comment une
surface redistribue la lumière incidente `ω_i` dans la direction observée `ω_o`. Elle a plusieurs
propriétés fondamentales :

**Réciprocité** : `f_r(ω_i, ω_o) = f_r(ω_o, ω_i)` — la lumière se propage de la même façon dans
les deux sens.

**Conservation de l'énergie** : `∫ f_r(ω_i, ω_o) × cos(θ_o) dω_o ≤ 1` — une surface ne peut
pas réfléchir plus de lumière qu'elle n'en reçoit.

**Non-négativité** : `f_r ≥ 0` partout.

Dans le path tracer, la BRDF intervient dans le calcul du rebond diffus et direct :
```
L_o = ∫ f_r(ω_i, ω_o) × L_i(ω_i) × cos(θ_i) dω_i
```

---

## 2. BRDF Lambertienne vs Cook-Torrance

### Lambertienne (actuelle)

```
f_r = albedo / π
```

Uniforme dans toutes les directions — bonne approximation pour les surfaces très rugueuses
(béton, tissu, bois), mais incapable de modéliser les reflets brillants.

### Cook-Torrance (microfacettes)

Les matériaux réels sont composés de microfacettes orientées aléatoirement. La BRDF
Cook-Torrance modélise :
- **D** : distribution des orientations des microfacettes (combien sont alignées avec le
  demi-vecteur `h = normalize(L + V)`)
- **F** : effet Fresnel — quelle fraction de la lumière est réfléchie vs absorbée
- **G** : géométrie — masquage/ombrage entre microfacettes voisines

```
f_specular(ω_i, ω_o) = D(h) × F(ω_o, h) × G(ω_i, ω_o) / (4 × (n·ω_i) × (n·ω_o))
```

La BRDF complète mélange un terme diffus (Lambertien) et un terme spéculaire :
```
f_r = k_d × (albedo / π)   +   k_s × f_specular
```
où `k_d = (1 - metalness) × (1 - F)` et `k_s = 1`.

Pour un métal (`metalness = 1`) : pas de composante diffuse, seulement du spéculaire.
Pour un diélectrique (`metalness = 0`) : les deux composantes contribuent.

---

## 3. Composante D — Normal Distribution Function (GGX)

La distribution GGX (Trowbridge-Reitz) est aujourd'hui le standard industriel :

```
D_GGX(n·h, α) = α² / (π × ((n·h)² × (α² - 1) + 1)²)
```

avec `α = roughness²` (convention Disney : roughness=0.5 donne une apparence mi-brillante).

**Interprétation** :
- `n·h = 1` (demi-vecteur aligné avec la normale) → pic maximum
- `n·h → 0` (lumière rasante) → D → 0
- `α = 1` (roughness=1) → distribution plate, D(1) = 1/π ≈ 0.318
- `α → 0` (roughness=0) → distribution très piquée, presque un Dirac

**Valeur remarquable** : `D_GGX(1.0, 1.0) = 1/π`

---

## 4. Composante G — Geometry (Smith GGX)

Le terme géométrique atténue la BRDF quand les microfacettes se masquent mutuellement.
Smith decompose G en deux termes indépendants :

```
G_Smith(n·l, n·v, α) = G1(n·l, α) × G1(n·v, α)

G1(n·x, α) = (n·x) / ((n·x) × (1 - k) + k)

k = α² / 2   (version IBL — recommandée)
```

**Propriétés** :
- `G ∈ [0, 1]`
- `G(1, 1, α) = 1` — angles normaux, pas d'ombrage
- G diminue aux angles rasants (n·l → 0 ou n·v → 0)
- k = α²/2 est la variante "smooth" adaptée à l'IBL et au path tracing

---

## 5. Composante F — Fresnel (Schlick)

L'effet Fresnel décrit comment la réflectance augmente aux angles rasants. Approximation de
Schlick :

```
F_Schlick(v·h, F0) = F0 + (1 - F0) × (1 - v·h)^5
```

`F0` est la réflectance à incidence normale — dépend de l'IOR (indice de réfraction) :

```
F0 = ((IOR - 1) / (IOR + 1))²
```

Valeurs typiques :
| Matériau      | IOR  | F0    |
|---------------|------|-------|
| Verre         | 1.5  | 0.04  |
| Eau           | 1.33 | 0.02  |
| Diamant       | 2.4  | 0.17  |
| Or (métal)    | —    | (0.95, 0.64, 0.37) |

Pour les métaux, `F0` est une couleur Vec3d définie par l'artiste via `albedo × metalness`.

---

## 6. Workflow metalness/roughness

Le modèle PBR à deux paramètres :

```
F0 = lerp(F0_from_ior(ior), albedo_linear, metalness)
     // diélectrique → F0 scalaire ; métal → F0 coloré (l'albedo)

k_d = (1 - metalness)    // pas de diffus pour les métaux

f_r = k_d × (albedo_linear / π)                        // diffus (Lambertien)
    + D(NdotH, α) × F(VdotH, F0) × G(NdotL, NdotV, α) // spéculaire Cook-Torrance
      / (4 × NdotL × NdotV)
```

où `NdotL = max(0, n·l)`, `NdotV = max(0, n·v)`, `NdotH = max(0, n·h)`,
`VdotH = max(0, v·h)`, `h = normalize(l + v)`.

---

## 7. C++ — namespace `physics` et closures multi-paramètres

### namespace imbriqué

Le namespace `rt::physics` isole les fonctions mathématiques pures (D, F, G) du reste du
renderer. Il n'y a aucune classe, aucun état — uniquement des fonctions stateless :

```cpp
namespace rt {
namespace physics {

inline double D_GGX(double NdotH, double roughness) { /* ... */ }
inline double G_Smith(double NdotL, double NdotV, double roughness) { /* ... */ }
inline Vec3d  F_Schlick(double VdotH, const Vec3d& F0) { /* ... */ }
inline double F0_from_ior(double ior) { /* ... */ }

}  // namespace physics
}  // namespace rt
```

Usage dans le renderer : `rt::physics::D_GGX(...)` ou `using namespace rt::physics;` localement.

**Avantage** : on peut tester chaque composante isolément, et le compilateur optimise mieux les
fonctions pures sans état.

### Closure multi-paramètres

`cook_torrance` est une factory capturant 4 paramètres physiques. Comparer avec Lambertian
(1 paramètre) :

```cpp
// Lambertien : 1 paramètre
auto lambertian = [](TexturePtr tex) -> Material {
    return [tex](const Ray&, const HitRecord& rec) -> optional<Scatter> { ... };
};

// Cook-Torrance : 4 paramètres
inline Material cook_torrance(Vec3d albedo, double roughness, double metalness, double ior) {
    double alpha = roughness * roughness;       // convention Disney
    Vec3d  F0    = Vec3d(F0_from_ior(ior));
    F0 = lerp(F0, albedo / 255.0, metalness);  // métal : F0 coloré

    return [albedo, alpha, metalness, F0](const Ray& r, const HitRecord& rec)
                -> optional<Scatter> {
        // ...
    };
}
```

Les captures `albedo`, `alpha`, `metalness`, `F0` sont copiées dans le lambda — chaque instance
de matériau est indépendante.

## 8. Sampling dans le path tracer

Pour retourner un `Scatter`, le matériau doit :
1. Choisir une direction `L` (le rayon rebondissant)
2. Calculer le **throughput** = `f_r × cos(θ) / pdf`
3. Retourner `Scatter{ throughput × 255, Ray(point, L) }`

### Option A — Cosine-weighted sampling (simple, ce TP)

```
L   = direction cosine-pondérée dans l'hémisphère (comme Lambertien)
pdf = cos(θ) / π
throughput = f_r × cos(θ) / pdf = f_r × π
attenuation = throughput × 255 (clampé à 255)
```

Simple, unbiasé, mais variance élevée pour les matériaux brillants.

### Option B — GGX importance sampling (extension)

Échantillonner directement depuis la distribution GGX :
1. Générer un demi-vecteur `h` selon D_GGX (formule d'inversion analytique)
2. Réfléchir `V` autour de `h` pour obtenir `L`
3. `pdf_GGX = D(NdotH) × NdotH / (4 × VdotH)`

Réduit considérablement le bruit sur les matériaux brillants. Voir la section Exercices.

## 9. Interface à créer

Crée `src/materials/brdf.h` :

```cpp
#pragma once
#include "core/vec3.h"
#include "geometry/shape.h"
#include <cmath>
#include <algorithm>

namespace rt {
namespace physics {

// Normal Distribution Function — GGX/Trowbridge-Reitz
// NdotH : cos de l'angle entre la normale et le demi-vecteur h ∈ [0, 1]
// roughness : paramètre artistique [0, 1], α = roughness² en interne
inline double D_GGX(double NdotH, double roughness);

// Geometry — Smith GGX
// NdotL, NdotV : cosinus des angles d'incidence et d'observation ∈ [0, 1]
inline double G_Smith(double NdotL, double NdotV, double roughness);

// Fresnel — Schlick
// VdotH : cos de l'angle entre V et le demi-vecteur h ∈ [0, 1]
// F0 : réflectance à incidence normale [0, 1]
inline Vec3d F_Schlick(double VdotH, const Vec3d& F0);

// F0 depuis l'IOR (pour diélectriques)
inline double F0_from_ior(double ior);

// BRDF complète Cook-Torrance
// albedo_linear : albedo en [0, 1] (diviser par 255 avant d'appeler)
// Retourne f_r en sr⁻¹ (peut être > 1 pour des surfaces très lisses)
Vec3d brdf_cook_torrance(
    const Vec3d& albedo_linear,
    double roughness,
    double metalness,
    double ior,
    const Vec3d& N,   // normale de surface (normalisée)
    const Vec3d& V,   // direction vers la caméra (normalisée)
    const Vec3d& L    // direction vers la lumière (normalisée)
);

}  // namespace physics

// Factory material
// albedo  : couleur de base en [0, 255]
// roughness : [0, 1]  — 0 = miroir, 1 = très mat
// metalness : [0, 1]  — 0 = diélectrique, 1 = métal pur
// ior       : indice de réfraction (1.5 = plastique/verre, 1.33 = eau)
inline Material cook_torrance(Vec3d albedo, double roughness,
                               double metalness, double ior = 1.5);

}  // namespace rt
```

## 10. Exercices

### Étape 1 — Composantes D, F, G

Dans `src/materials/brdf.h`, implémente les 4 fonctions du namespace `physics` :

**D_GGX** :
```
alpha = roughness²
dénominateur = π × ((NdotH² × (alpha² - 1) + 1)²)
D = alpha² / dénominateur
```

**G_Smith** :
```
k  = alpha² / 2
G1(NdotX) = NdotX / (NdotX × (1-k) + k)
G_Smith = G1(NdotL) × G1(NdotV)
```

**F_Schlick** :
```
F = F0 + (1 - F0) × (1 - VdotH)^5
```
(composante par composante sur Vec3d)

**F0_from_ior** :
```
r = (ior - 1) / (ior + 1)
F0 = r × r
```

**Vérifie** avec les tests unitaires (section D, F, G dans test_pbr.cpp).

### Étape 2 — brdf_cook_torrance

Implémente la BRDF complète :

```
h = normalize(L + V)
NdotH = max(0, dot(N, h))
NdotL = max(0, dot(N, L))
NdotV = max(0, dot(N, V))
VdotH = max(0, dot(V, h))

alpha = roughness²
F0 = lerp(Vec3d(F0_from_ior(ior)), albedo_linear, metalness)

D = D_GGX(NdotH, roughness)
F = F_Schlick(VdotH, F0)
G = G_Smith(NdotL, NdotV, roughness)

f_specular = D × F × G / max(4 × NdotL × NdotV, 1e-7)

k_d = (1 - metalness) × (1 - F)   // F est la moyenne de F ici, ou composante X
f_diffuse = k_d × (albedo_linear / π)

return f_diffuse + f_specular
```

> Astuce : pour `lerp(a, b, t)`, calcule `a * (1-t) + b * t`. Vec3d n'a pas de
> `lerp` intégré — tu peux utiliser l'arithmétique vectorielle.

### Étape 3 — Material cook_torrance

Implémente la factory `cook_torrance` qui retourne un `Material` :

1. Calculer `alpha = roughness²` et `F0` dans la closure externe (une seule fois)
2. Dans le lambda : générer une direction cosine-pondérée (comme Lambertien)
3. Calculer L et V depuis le rayon incident et le hit record
4. Appeler `brdf_cook_torrance(...)` pour obtenir `f_r`
5. Calculer le throughput = `f_r × π` (pour cosine sampling, pdf = cos(θ)/π)
6. Retourner `Scatter{ clamp(throughput × 255, 0, 255), Ray(point, L_sampled) }`

### Étape 6 — GGX Importance Sampling (extension)

Pour réduire le bruit sur les matériaux brillants, remplace l'échantillonnage cosine-pondéré
par l'inversion analytique de D_GGX :

```
xi1, xi2 : deux nombres aléatoires dans [0, 1]

phi   = 2π × xi2
cos_theta = sqrt((1 - xi1) / (1 + (alpha² - 1) × xi1))
sin_theta = sqrt(1 - cos_theta²)

h_local = (sin_theta × cos(phi), sin_theta × sin(phi), cos_theta)
h_world = transformer h_local dans l'espace de la normale (comme TP19)
L       = reflect(-V, h_world)    // symétrie par rapport à h

pdf_GGX = D_GGX(NdotH, roughness) × NdotH / (4 × max(VdotH, 1e-7))
throughput = F × G × VdotH / (NdotV × NdotH + 1e-7)
```

Comparer le bruit entre cosine-sampling et GGX-sampling pour `silver_pbr` à 32 samples.
