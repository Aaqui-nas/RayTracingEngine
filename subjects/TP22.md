Jusqu'ici toutes tes primitives (`Sphere`, `Plane`, `Triangle`) étaient définies manuellement.  
Dans ce TP tu vas **générer des maillages** (`Mesh`) de façon procédurale — le code produit les sommets et les triangles à partir d'une formule mathématique ou d'une grille de hauteurs.

C++ introduit : les algorithmes générateurs de la STL.

## `std::generate` — rappel rapide

```cpp
std::vector<double> v(10);
int n = 0;
std::generate(v.begin(), v.end(), [&n]() { return n++ * 0.1; });
// v == { 0.0, 0.1, 0.2, ..., 0.9 }
```

Le troisième argument est un **callable sans paramètre** appelé une fois par élément.  
L'état se porte dans la capture du lambda.

`std::generate_n(it, count, gen)` fait la même chose mais écrit `count` éléments à partir de `it`.

## Exercice 1 — `std::generate` sur une grille (30 min)

Avant d'attaquer les maillages, maîtrise le pattern.

**Signature :**
```cpp
// Retourne un vecteur de nu*nv Vec3d.
// Chaque élément est la position (u/nu, v/nv, 0) dans [0,1]².
std::vector<Vec3d> make_uv_grid(int nu, int nv);
```

**Algorithme :**
1. Crée un `vector<Vec3d>` de taille `nu * nv`
2. Utilise `std::generate` avec un lambda qui maintient un compteur `k`
3. Pour l'élément `k` : `u = k / nv`, `v = k % nv` (division entière)
4. Retourne `Vec3d(double(u)/nu, double(v)/nv, 0)`

**Vérification :** `make_uv_grid(4, 3)[7]` → `Vec3d(0.5, 0.333..., 0)` (u=2/4, v=1/3).

## Exercice 2 — Génération d'un tore (90 min)

Un tore de grand rayon R et petit rayon r.  
Paramétré par deux angles u ∈ [0, 2π] et v ∈ [0, 2π].

**Formule d'un sommet :**
```
x = (R + r·cos(v)) · cos(u)
y =  r · sin(v)
z = (R + r·cos(v)) · sin(u)
```

**Formule de la normale (sortante) :**
```
nx = cos(u) · cos(v)
ny = sin(v)
nz = sin(u) · cos(v)
```

**Discrétisation :**
- `nu` tranches autour du grand cercle (axe Y), `nv` tranches autour du petit tube
- Angle : `u_i = 2π · i / nu`, `v_j = 2π · j / nv`
- Indice linéaire : `k = i * nv + j`

**Algorithme :**

```
1. Génère les nu*nv sommets avec std::generate :
   Pour l'élément k : i = k/nv, j = k%nv → calcule le Vec3d avec les formules ci-dessus

2. Pour chaque quad (i, j) → 2 triangles :
   - A = vertex(i,   j)      indice = i*nv + j
   - B = vertex(i+1, j)      indice = ((i+1)%nu)*nv + j
   - C = vertex(i,   j+1)    indice = i*nv + ((j+1)%nv)
   - D = vertex(i+1, j+1)    indice = ((i+1)%nu)*nv + ((j+1)%nv)
   Triangle 1 : A, B, C
   Triangle 2 : B, D, C

3. Ajoute chaque triangle au Mesh avec mesh.add(Triangle(A, B, C, mat))
```

**Counts attendus :** `nu*nv` sommets, `2*nu*nv` triangles.

## Exercice 3 — Terrain depuis une grille de hauteurs (60 min)

Un terrain est une grille `N×M` où chaque case a une hauteur.  
`heights[i][j]` = hauteur de la colonne i, ligne j.

**Vertex (i, j) :**
```
x = i * cell_size
y = heights[i][j] * height_scale
z = j * cell_size
```

**Algorithme :**

```
1. N = heights.size(), M = heights[0].size()

2. Génère les N*M sommets avec std::generate_n :
   lambda maintient i et j, calcule Vec3d(i*cell_size, heights[i][j]*height_scale, j*cell_size)
   passe au suivant : j++, si j==M alors j=0, i++

3. Pour chaque quad (i, j) avec i < N-1, j < M-1 → 2 triangles :
   - A = vertex(i,   j)    indice = i*M + j
   - B = vertex(i+1, j)    indice = (i+1)*M + j
   - C = vertex(i,   j+1)  indice = i*M + (j+1)
   - D = vertex(i+1, j+1)  indice = (i+1)*M + (j+1)
   Triangle 1 : A, C, B
   Triangle 2 : B, C, D

4. Ajoute chaque triangle au Mesh
```

**Counts attendus :** `N*M` sommets, `2*(N-1)*(M-1)` triangles.

## Exercice 4 — Sphere mesh (30 min)

`Sphere` existe déjà comme primitive implicite, mais un `Mesh` de sphère permet d'appliquer des normal maps, des textures UV, ou des déformations procédurales.

**Paramétrique :**
```
x = radius · sin(theta) · cos(phi)
y = radius · cos(theta)
z = radius · sin(theta) · sin(phi)

theta_i = π · i / (nu-1)     // 0 → π
phi_j   = 2π · j / nv        // 0 → 2π
```

**UV :**
```
u = phi_j / (2π)
v = theta_i / π
```

**Algorithme :**

```
1. Génère les nu*nv sommets (même structure que le tore, mais avec les formules ci-dessus)
2. Les quads : attention, les pôles (i=0 et i=nu-1) sont dégénérés → triangle unique au lieu de quad
3. Pour les lignes i ∈ [0, nu-2] : génère les quads normalement
4. (Bonus) Gère les pôles avec des triangles dégénérés
```

## Exercice 5 (bonus) — Déformation procédurale (40 min)

Une fois le terrain généré, applique un bruit de Perlin pour le déformer.

**Signature :**
```cpp
// Modifie heights en place : heights[i][j] += perlin.noise(i*scale, j*scale)
void apply_noise(std::vector<std::vector<double>>& heights,
                 double scale, double amplitude);
```

**Algorithme :**
```
1. Pour chaque (i, j) :
   heights[i][j] += amplitude * perlin_noise(i*scale, j*scale)
   (le bruit est dans [-1, 1], amplitude le module)
```

`Perlin` est déjà dans `src/materials/perlin.h`.
