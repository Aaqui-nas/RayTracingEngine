# TP30 — Participating Media : brouillard et fumée

**Durée estimée : ~12h**
**C++ introduit :** Intégration numérique (somme de Riemann), random walk Monte Carlo, `std::uniform_real_distribution`
**Feature graphique :** Brouillard homogène, fumée hétérogène, raymarching

---

## Objectif

Jusqu'ici, les rayons voyagent dans le vide entre les surfaces. Dans la réalité, les milieux participatifs — brouillard, fumée, nuages, eau trouble — **absorbent et diffusent** la lumière le long du chemin parcouru.

Ce TP introduit les **participating media** (milieux participatifs) via deux approches :
1. **Raymarching déterministe** : avancer le long du rayon à pas fixes et accumuler la lumière
2. **Random walk Monte Carlo** : choisir aléatoirement si et où le rayon interagit avec le medium

---

## 1. Modèle physique : Beer-Lambert

La loi de Beer-Lambert décrit comment la lumière est atténuée en traversant un medium homogène :

```
T(d) = exp(-sigma_t × d)
```

- `T(d)` : transmittance — fraction de lumière restante après avoir traversé une distance `d`
- `sigma_t` : coefficient d'extinction total (absorption + diffusion)
  - `sigma_a` : coefficient d'absorption (lumière convertie en chaleur)
  - `sigma_s` : coefficient de diffusion (lumière redirigée dans une autre direction)
  - `sigma_t = sigma_a + sigma_s`

**Albedo de diffusion unique :**
```
omega = sigma_s / sigma_t   ∈ [0, 1]
```
- `omega = 0` : medium purement absorbant (encre noire)
- `omega = 1` : medium purement diffusant (nuage blanc)
- `omega = 0.5` : fumée typique

---

## 2. Homogène vs hétérogène

**Medium homogène** : `sigma_t` constant partout → Beer-Lambert direct, solution analytique.

**Medium hétérogène** : `sigma_t(x, y, z)` varie dans l'espace — nécessite l'intégration numérique.

```
T(d) = exp(-∫₀ᵈ sigma_t(x(s)) ds)
```

Cette intégrale n'a généralement pas de forme close → on la calcule numériquement.

---

## 3. Raymarching

**Principe :** avancer le long du rayon à pas réguliers `delta_s`, accumuler l'émission et l'atténuation.

```
transmittance = 1.0
color = (0, 0, 0)

Pour s = s_start, s + delta_s, ..., s_end :
    position = ray_origin + s × ray_direction
    sigma = density(position) × sigma_t_max
    
    // Accumulation émission
    L_scatter = sigma_s × phase(dir_entrant, dir_sortant) × L_environment
    color += transmittance × L_scatter × delta_s
    
    // Atténuation
    transmittance *= exp(-sigma × delta_s)
    
    if transmittance < 1e-5 : break   // early termination

// Couleur de fond atténuée
color += transmittance × background_color
```

---

## 4. Random Walk (Delta Tracking)

Au lieu de marcher à pas fixes, on choisit aléatoirement **où** le rayon interagit. C'est plus efficace pour les milieux hétérogènes.

**Algorithme Woodcock (Delta Tracking) :**

```
sigma_max = borne supérieure de sigma_t dans le volume

t = t_start
while t < t_end :
    // Tire la prochaine distance d'interaction
    xi = random_uniform(0, 1)
    t -= log(xi) / sigma_max   // distribution exponentielle

    if t >= t_end : break   // rayon sort du volume

    // Acceptation/rejet
    sigma_at_t = density(ray(t)) × sigma_t_max
    if random_uniform(0, 1) < sigma_at_t / sigma_max :
        // Vraie interaction à t — diffusion ou absorption
        if random_uniform(0, 1) < omega :
            // Diffusion : génère nouvelle direction
            retourner t et la nouvelle direction
        else :
            // Absorption : rayon arrêté
            retourner t, atténuation = 0
```

L'avantage : on saute automatiquement les zones vides — beaucoup plus efficace que raymarching à pas fixes.

## Exercice 1 — Medium homogène (60 min)

Crée `src/materials/volume.h`. Un `HomogeneousVolume` est une `Shape` sphérique ou AABB remplie d'un medium uniforme.

```cpp
class ConstantMedium : public Shape {
    std::shared_ptr<Shape> boundary;   // contour du volume (sphère, box...)
    double neg_inv_density;            // = -1 / sigma_t
    Vec3d phase_color;                 // couleur du milieu

public:
    ConstantMedium(std::shared_ptr<Shape> boundary,
                   double density, Vec3d color);

    std::optional<HitRecord> hit(const Ray& r, double tmin, double tmax) const override;
    std::optional<AABB> bounding_box() const override;
};
```

**Algorithme de `hit` pour ConstantMedium :**

```
1. Trouve les deux intersections avec boundary : t1 (entrée) et t2 (sortie)
   (boundary->hit pour t1, puis pour t2 depuis t1+epsilon)

2. Tire la distance d'interaction dans le medium :
   xi = random_uniform(0, 1)
   distance_in_medium = neg_inv_density × log(xi)
                         = (-1/sigma_t) × log(xi)

3. Si distance_in_medium > (t2 - t1) : le rayon traverse sans interagir
   return nullopt

4. Sinon :
   t_hit = t1 + distance_in_medium
   point = ray(t_hit)
   normal = (1, 0, 0)   // arbitraire pour les volumes
   
   HitRecord rec;
   rec.t = t_hit;
   rec.point = point;
   rec.normal = Vec3d(1, 0, 0);   // convention
   rec.front_face = true;
   rec.material = isotropic_material(phase_color);
   return rec;
```

**Material `isotropic` :**

Un matériau isotrope diffuse dans une direction **uniformément aléatoire** (sphère entière, pas hémisphère) :

```cpp
Material isotropic(Vec3d color) {
    return [color](const Ray& r_in, const HitRecord& rec) -> std::optional<Scatter> {
        Vec3d direction = random_unit_sphere();   // direction aléatoire dans la sphère complète
        return Scatter{ color, Ray(rec.point, direction) };
    };
}
```

## Exercice 3 — Rendu avec raymarching déterministe (90 min)

Implémente un raymarcher dans le path tracer. L'idée : avant de retourner la couleur d'un rayon, s'il traverse une zone volumétrique, accumule l'émission diffuse le long du chemin.

Crée `src/materials/raymarcher.h` :

```cpp
struct VolumeGrid {
    std::vector<float> density;   // densité en chaque voxel
    int nx, ny, nz;               // résolution de la grille
    Vec3d bounds_min, bounds_max;  // AABB du volume

    float density_at(const Vec3d& p) const;
    // retourne la densité au point p par interpolation trilinéaire
};

Vec3d raymarch(
    const Ray& ray,
    const VolumeGrid& grid,
    double sigma_t, double sigma_s,
    int num_steps,
    double t_start, double t_end);
```

**Interpolation trilinéaire pour la densité :**

```
Convertit p en coordonnées grille : ix = (p.x - bounds_min.x) / cell_size
Prend les 8 coins voisins (ix, ix+1) × (iy, iy+1) × (iz, iz+1)
Interpole bilinéairement d'abord sur X, puis Y, puis Z
```

**Génération procédurale de densité :**

Utilise du bruit de Perlin pour générer la grille de densité :
```cpp
VolumeGrid generate_smoke(int nx, int ny, int nz, double noise_scale, double density_scale) {
    VolumeGrid grid{/* ... */};
    for (int k = 0; k < nz; k++)
        for (int j = 0; j < ny; j++)
            for (int i = 0; i < nx; i++) {
                double d = perlin.noise(i*noise_scale, j*noise_scale, k*noise_scale);
                d = std::max(0.0, d);   // ne garde que les valeurs positives
                grid.density[k*ny*nx + j*nx + i] = (float)(d * density_scale);
            }
    return grid;
}
```

## Exercice 4 — Delta tracking pour volume hétérogène (90 min)

Implémente le Woodcock sampling pour les milieux non-uniformes :

```cpp
std::optional<double> delta_tracking(
    const Ray& ray,
    const VolumeGrid& grid,
    double sigma_t_max,   // borne supérieure de sigma_t
    double t_start, double t_end,
    std::mt19937& rng);
```

**Retourne :** `t` de la prochaine interaction si elle a lieu dans `[t_start, t_end]`, sinon `nullopt`.

**Algorithme :**
```
t = t_start
while true :
    xi = uniform(rng)
    t -= log(xi) / sigma_t_max
    if t >= t_end : return nullopt

    pos = ray(t)
    sigma_local = grid.density_at(pos) × sigma_t_max
    
    if uniform(rng) < sigma_local / sigma_t_max :
        return t   // interaction à t
```

**Avantage vs raymarching :** delta tracking s'adapte automatiquement à la densité — il avance vite dans les zones vides et lentement dans les zones denses.

## Exercice 5 — Phase function (30 min)

La **fonction de phase** décrit comment la lumière est redistribuée lors d'une diffusion.

**Isotrope :** `p(θ) = 1/(4π)` — uniforme dans toutes les directions.

**Henyey-Greenstein :** modèle standard pour la fumée et les nuages :

```
p(θ, g) = (1 - g²) / (4π × (1 + g² - 2g·cos(θ))^(3/2))
```

- `g = 0` : isotrope
- `g > 0` : forward scattering (lumière qui continue dans la même direction — nuages)
- `g < 0` : backward scattering (lumière renvoyée vers la source — ketchup)

**Sampling Henyey-Greenstein :**

```
xi = uniform(0, 1)
cos_theta = (1 + g² - ((1-g²)/(1-g+2g×xi))²) / (2g)   si g ≠ 0
cos_theta = 1 - 2*xi                                      si g = 0
```

Implémente `sample_hg(double g, Vec3d dir_in)` qui retourne une direction de diffusion.
