# TP11 — Move Semantics et Optimisation

## Objectif final
Comprendre comment C++ évite les copies inutiles, et optimiser le passage d'objets dans le ray tracer.

---

## 1. Lvalue et Rvalue

- **lvalue** : un objet nommé, qui a une adresse → `Vec3 v = ...;`
- **rvalue** : une valeur temporaire, sans nom → `Vec3(1, 2, 3)`, résultat d'une fonction

```cpp
Vec3 v(1, 2, 3);   // v est une lvalue
v + Vec3(0,0,1);   // Vec3(0,0,1) est une rvalue
```

---

## 2. Constructeur de copie vs constructeur de mouvement

```cpp
class Buffer {
    int* data;
    int  size;
public:
    // Copie : alloue et copie les données
    Buffer(const Buffer& other) : size(other.size) {
        data = new int[size];
        std::copy(other.data, other.data + size, data);
    }

    // Mouvement : vole les données, sans copie
    Buffer(Buffer&& other) noexcept : data(other.data), size(other.size) {
        other.data = nullptr;  // l'ancien objet ne doit plus libérer la mémoire
        other.size = 0;
    }
};
```

---

## 3. `std::move`

`std::move` caste une lvalue en rvalue pour déclencher le constructeur de mouvement :

```cpp
Buffer a = make_big_buffer();
Buffer b = std::move(a);  // a est maintenant vide, aucune copie
```

---

## 4. Perfect forwarding avec `&&`

```cpp
void add_object(std::shared_ptr<Shape>&& shape) {
    objects.push_back(std::move(shape));
}
```
