# TP32 — LOD et Instancing

**Durée estimée : ~6h**
**C++ introduit :** Proxy objects, `shared_ptr` partagé entre instances, index de LOD, `std::weak_ptr`
**Feature graphique :** 1000 instances d'un mesh sans ×1000 la mémoire, LOD automatique selon la distance

---

## Objectif

Une scène de jeu typique peut avoir des milliers d'arbres, de rochers, d'ennemis. Si chaque objet possède **sa propre copie** du mesh, la mémoire explose.

Ce TP introduit deux techniques fondamentales des moteurs de jeu :

1. **Instancing** — partager le même mesh entre N objets, seule la transformation est dupliquée
2. **LOD (Level of Detail)** — utiliser un mesh plus simple pour les objets distants

Ces techniques sont universelles : Unreal Engine, Unity, Godot les utilisent tous.

---

## 1. Le problème sans instancing

```
Scène sans instancing :
  Arbre 1 : Mesh (10 000 triangles) + Transform
  Arbre 2 : Mesh (10 000 triangles) + Transform  ← copie complète
  Arbre 3 : Mesh (10 000 triangles) + Transform  ← encore
  ...
  Arbre 1000 : ...
  Total : 10 000 000 triangles en mémoire
```

```
Scène avec instancing :
  MeshAsset arbre : Mesh (10 000 triangles)  ← une seule copie
  Arbre 1 : référence vers arbre + Transform
  Arbre 2 : référence vers arbre + Transform  ← pas de copie
  ...
  Total : 10 000 triangles + 1000 transforms
```

---

## 2. Instancing avec `shared_ptr`

`std::shared_ptr<Mesh>` permet exactement ce partage : plusieurs objets pointent vers le même mesh, la mémoire est libérée quand le dernier pointeur disparaît.

```cpp
auto mesh = std::make_shared<Mesh>(load_mesh("tree.obj"));  // chargé une fois

for (int i = 0; i < 1000; i++) {
    auto transform = Mat4::translate(random_x(), 0, random_z());
    scene.add(std::make_shared<MeshInstance>(mesh, transform));
}
// mesh est référencé 1000 fois, stocké une seule fois
```

---

## 3. LOD — Level of Detail

L'idée : utiliser des meshes de différentes résolutions selon la distance caméra.

```
Distance 0–5 m   : LOD0 — mesh haute résolution (10 000 triangles)
Distance 5–20 m  : LOD1 — mesh moyenne résolution (1 000 triangles)
Distance 20–100 m : LOD2 — mesh basse résolution (100 triangles)
Distance > 100 m : LOD3 — billbord ou invisible
```

Généralement, LOD0 est le mesh original, LOD1-3 sont des versions simplifiées.

## Exercice 1 — MeshInstance (45 min)

Crée `src/geometry/mesh_instance.h`. Un `MeshInstance` est un proxy qui applique une transformation à un mesh partagé :

```cpp
class MeshInstance : public Shape {
    std::shared_ptr<Mesh> mesh;   // partagé
    Mat4 local_to_world;
    Mat4 world_to_local;
    Mat4 normal_matrix;

public:
    MeshInstance(std::shared_ptr<Mesh> mesh, const Mat4& transform);

    std::optional<HitRecord> hit(const Ray& r, double tmin, double tmax) const override;
    std::optional<AABB> bounding_box() const override;

    // Accès à la transformation (pour mettre à jour la scène)
    void set_transform(const Mat4& transform);
    const Mat4& transform() const;
};
```

**`hit` :**  
Même logique que `TransformNode` du TP25 : transformer le rayon en espace local, déléguer à `mesh->hit`, transformer le résultat.

**`bounding_box` :**  
Prend les 8 coins de l'AABB du mesh en espace local, les transforme dans l'espace monde, retourne l'AABB englobante des 8 coins transformés.

## Exercice 3 — AssetCache (45 min)

Dans une vraie application, on ne veut pas recharger le même fichier mesh plusieurs fois.

Crée `src/scene/asset_cache.h` :

```cpp
class AssetCache {
    std::unordered_map<std::string, std::weak_ptr<Mesh>> cache;
    std::mutex mutex;

public:
    std::shared_ptr<Mesh> load_mesh(const std::string& path);
};
```

**`load_mesh` :**

```
1. Cherche path dans cache
2. Si trouvé et shared_ptr valide (weak_ptr.lock() != nullptr) :
      return le shared_ptr existant
3. Sinon :
      charge le mesh depuis le fichier
      stocke un weak_ptr dans le cache
      return le shared_ptr
```

**Pourquoi `weak_ptr` dans le cache ?**  
Si on stockait un `shared_ptr`, le cache empêcherait la libération de la mémoire même quand plus personne n'utilise le mesh. `weak_ptr` ne prolonge pas la durée de vie — si toutes les instances sont supprimées, le mesh est libéré et le cache retourne `nullptr` à la prochaine requête.

## Exercice 4 — LOD group (60 min)

Crée `src/geometry/lod_group.h` :

```cpp
struct LODLevel {
    double min_distance;  // utiliser ce LOD si la caméra est à > min_distance
    std::shared_ptr<Shape> geometry;
};

class LODGroup : public Shape {
    std::vector<LODLevel> lods;    // trié par min_distance croissant
    Vec3d position;                // centre du groupe (pour calculer la distance)

public:
    LODGroup(Vec3d position);
    void add_lod(double min_distance, std::shared_ptr<Shape> geom);

    std::optional<HitRecord> hit(const Ray& r, double tmin, double tmax) const override;
    std::optional<AABB> bounding_box() const override;

    // Le LOD actif est sélectionné au moment du hit
    // Le ray contient l'origine caméra
};
```

**Sélection du LOD dans `hit` :**

```
distance_camera = length(r.origin - position)

Pour chaque LODLevel de la fin vers le début :
    si distance_camera >= lod.min_distance :
        return lod.geometry->hit(r, tmin, tmax)

return nullopt   // trop proche de aucun LOD (ne devrait pas arriver)
```

Typiquement :
```cpp
lod_group.add_lod(0.0, high_res_mesh);    // à partir de 0 : haute résolution
lod_group.add_lod(10.0, medium_mesh);     // à partir de 10 : moyen
lod_group.add_lod(50.0, low_mesh);        // à partir de 50 : basse résolution
lod_group.add_lod(200.0, nullptr);        // à partir de 200 : invisible
```

## Exercice 5 — Génération procédurale de tores LOD (45 min)

Utilise `generate_torus` du TP22 pour créer 3 niveaux de LOD :

```cpp
auto create_torus_lod(Material mat) {
    auto lod = std::make_shared<LODGroup>(Vec3d(0, 0, 0));

    lod->add_lod(0.0,
        std::make_shared<Mesh>(generate_torus(1.5, 0.5, 128, 64, mat)));  // haute res
    lod->add_lod(5.0,
        std::make_shared<Mesh>(generate_torus(1.5, 0.5, 32, 16, mat)));   // moyenne
    lod->add_lod(20.0,
        std::make_shared<Mesh>(generate_torus(1.5, 0.5, 8, 4, mat)));     // basse
    lod->add_lod(100.0, nullptr);  // invisible

    return lod;
}
```

## Exercice 6 — Scène de démonstration (30 min)

Crée une scène avec 100 tores instanciés à différentes distances :

```cpp
// Dans main.cpp ou dans un scene loader étendu
AssetCache cache;

// Créer 3 meshes LOD partagés
auto torus_hi  = cache.load_or_create("torus_hi",  []{
    return generate_torus(1.5, 0.5, 128, 64, materials["gold_pbr"]); });
auto torus_med = cache.load_or_create("torus_med", []{
    return generate_torus(1.5, 0.5, 32, 16, materials["gold_pbr"]); });
auto torus_lo  = cache.load_or_create("torus_lo",  []{
    return generate_torus(1.5, 0.5, 8, 4, materials["gold_pbr"]); });

for (int i = 0; i < 100; i++) {
    auto lod = std::make_shared<LODGroup>(random_position(30.0f));
    lod->add_lod(0.0,  std::make_shared<MeshInstance>(torus_hi,  translate(lod->position)));
    lod->add_lod(8.0,  std::make_shared<MeshInstance>(torus_med, translate(lod->position)));
    lod->add_lod(25.0, std::make_shared<MeshInstance>(torus_lo,  translate(lod->position)));
    lod->add_lod(80.0, nullptr);
    scene.add(lod);
}
```

**Compare le temps de rendu :**
- Sans LOD : 100 × 128×64 = 819 200 triangles par objet
- Avec LOD : les objets distants ont 8×4 triangles → dramatiquement moins de travail

```bash
./build/raytracer 1.0 64 scenes/scene_instances.txt > outputs/instances.ppm
```

## Critères de réussite

- [ ] `MeshInstance` : la même mesh partagée entre 100 instances, `sizeof(mesh) == 1 × taille_mesh`
- [ ] `mesh.use_count()` après 100 instances = 101 (100 instances + 1 référence originale)
- [ ] `AssetCache` : charger le même fichier deux fois retourne le même `shared_ptr`
- [ ] `AssetCache` : après suppression de toutes les instances, `weak_ptr.lock() == nullptr`
- [ ] `LODGroup` : les tores proches sont plus détaillés que les lointains (visible dans le rendu)
- [ ] Rendu de 100 instances avec LOD : plus rapide que 100 instances haute résolution
