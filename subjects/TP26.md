## Objectif

Le moteur de templates C++ est Turing-complet. Ce TP te fait explorer ses mécanismes de métaprogrammation :

- **SFINAE** — Substitution Failure Is Not An Error : activer ou désactiver des surcharges selon le type
- **`if constexpr`** — branchement à la compilation (C++17)
- **Type traits** — interroger les propriétés d'un type à la compilation (`std::is_floating_point_v`, etc.)
- **Tag dispatch** — sélectionner une implémentation via un type "tag" vide

Ces outils permettent d'écrire une **primitive `Box<T>`** générique qui fonctionne aussi bien avec `float` qu'avec `double`, et une **CSG** qui compose géométriquement n'importe quelles deux shapes.

## 1. SFINAE — Substitution Failure Is Not An Error

### Le problème

Supposons que tu veuilles écrire deux surcharges d'une fonction `serialize` : une pour les types flottants, une pour les entiers.

```cpp
// Version naïve — ne compile pas : ambiguïté
template<typename T>
void serialize(T val) { /* pour les flottants */ }

template<typename T>
void serialize(T val) { /* pour les entiers */ }
```

### La solution SFINAE

```cpp
#include <type_traits>

// Active cette surcharge SEULEMENT si T est un type flottant
template<typename T>
std::enable_if_t<std::is_floating_point_v<T>> serialize(T val) {
    std::cout << std::fixed << val;
}

// Active cette surcharge SEULEMENT si T est un entier
template<typename T>
std::enable_if_t<std::is_integral_v<T>> serialize(T val) {
    std::cout << std::hex << val;
}
```

`std::enable_if_t<condition>` retourne `void` si `condition` est vraie, et **ne compile pas** (substitution failure) si elle est fausse. Quand la substitution échoue, le compilateur **essaie la surcharge suivante** plutôt que d'émettre une erreur — d'où le nom SFINAE.

### SFINAE dans le type de retour vs paramètre

Deux placements courants :

```cpp
// Dans le type de retour (le plus classique)
template<typename T>
std::enable_if_t<std::is_floating_point_v<T>, Vec3d>
lerp(T a, T b, T t) { ... }

// Comme paramètre fictif (pratique quand le type de retour est complexe)
template<typename T, std::enable_if_t<std::is_floating_point_v<T>, int> = 0>
Vec3d lerp(T a, T b, T t) { ... }
```

## 2. `if constexpr`

C++17 introduit `if constexpr` — un branchement évalué **à la compilation** :

```cpp
template<typename T>
std::string type_name() {
    if constexpr (std::is_integral_v<T>)
        return "integer";
    else if constexpr (std::is_floating_point_v<T>)
        return "float";
    else
        return "other";
}
```

La branche non sélectionnée n'est **pas compilée** — elle peut contenir du code invalide pour ce type :

```cpp
template<typename T>
void maybe_sqrt(T val) {
    if constexpr (std::is_floating_point_v<T>) {
        std::cout << std::sqrt(val);   // ne compile que pour float/double
    }
}
maybe_sqrt(42);    // OK — branche sqrt jamais compilée pour int
maybe_sqrt(3.14);  // OK
```

Contrairement à un `if` ordinaire, `if constexpr` permet ce code même si la branche est invalide pour le type donné.

## 3. Type traits importants

| Trait | Vrai si |
|-------|---------|
| `std::is_floating_point_v<T>` | T est float, double ou long double |
| `std::is_integral_v<T>` | T est int, long, bool, char... |
| `std::is_same_v<T, U>` | T et U sont le même type |
| `std::is_base_of_v<Base, Derived>` | Derived hérite de Base |
| `std::is_pointer_v<T>` | T est un pointeur |
| `std::is_const_v<T>` | T est const-qualifié |
| `std::decay_t<T>` | T sans const/ref/array decay |
| `std::remove_cvref_t<T>` | T sans const, volatile, référence (C++20) |
| `std::conditional_t<B, T, F>` | T si B, F sinon |

## Exercice 2 — Box axis-aligned avec SFINAE (60 min)

Crée `src/geometry/box.h`. Une boîte axis-aligned est définie par ses deux coins `pmin` et `pmax`.

**Intersection rayon/AABB (algorithme slab) :**

```
Pour chaque axe i ∈ {x, y, z} :
    t_near_i = (pmin[i] - origin[i]) / direction[i]
    t_far_i  = (pmax[i] - origin[i]) / direction[i]
    Si direction[i] < 0 : swap(t_near_i, t_far_i)

t_enter = max(t_near_x, t_near_y, t_near_z)
t_exit  = min(t_far_x,  t_far_y,  t_far_z)

hit si : t_enter <= t_exit && t_exit > tmin && t_enter < tmax
```

**Normale au point d'intersection :**  
Identifie quelle face a été touchée (quelle paire slab est responsable de `t_enter`) — c'est l'axe où `t_near == t_enter`. La normale est ±e_i (vecteur de base) selon si on touche la face `pmin` ou `pmax`.

**Classe :**
```cpp
class Box : public Shape {
    Vec3d pmin, pmax;
public:
    Box(Vec3d pmin, Vec3d pmax, Material mat);

    std::optional<HitRecord> hit(const Ray& r, double tmin, double tmax) const override;
    std::optional<AABB> bounding_box() const override;

    // UV mapping sur les 6 faces (bonus)
    Vec2d uv_at(const Vec3d& point, const Vec3d& normal) const;
};
```

Utilise SFINAE pour fournir deux constructeurs :
- Si `pmin` et `pmax` sont déjà triés composante par composante : constructeur direct
- Sinon : un constructeur factory qui les réordonne

## Exercice 3 — Cylindre et Cône (90 min)

**Cylindre** (axe Y, rayon r, hauteur h) :

L'intersection d'un rayon avec le corps cylindrique résout :
```
dx² + dz² = r²  (cercle en XZ)
```

Avec `oc = origin - center` :
```
a = dx² + dz²
b = 2(oc.x*dx + oc.z*dz)
c = oc.x² + oc.z² - r²
discriminant = b² - 4ac
```

Deux racines → prendre la plus petite dans [tmin, tmax] ET vérifier que `y ∈ [0, h]`.

Gérer les **caps** (disques supérieur et inférieur) séparément :
- Intersecte le rayon avec les plans `y=0` et `y=h`
- Vérifie que `x² + z² ≤ r²` au point d'intersection

**Cône** (apex à l'origine, axe Y, demi-angle α) :

L'équation implicite est `x² + z² = (y·tan(α))²` pour `y ∈ [0, h]`.

```
a = dx² + dz² - dtan²·dy²
b = 2(oc.x*dx + oc.z*dz - oc.y*dtan²*dy)
c = oc.x² + oc.z² - (oc.y*dtan)²
```

Avec `dtan = tan(half_angle)`.

**Utilise `if constexpr` pour factoriser :**

Cylindre et Cône partagent la même structure d'intersection (discriminant, deux racines, test de hauteur). Tu peux écrire un template interne :

```cpp
template<bool IsCone>
static std::optional<HitRecord> intersect_quadric(
    const Ray& r, double tmin, double tmax,
    double radius, double height,
    double half_angle = 0.0)
{

    } else {
        // formules cylindre
```

## Exercice 4 — CSG : Constructive Solid Geometry (90 min)

La CSG permet de composer des volumes :
- **Union** : l'intérieur de A OU de B
- **Intersection** : l'intérieur de A ET de B  
- **Différence** : l'intérieur de A MAIS PAS de B

**Algorithme de hit CSG :**

Pour un rayon, collecte **tous** les intervalles d'entrée/sortie de chaque objet, puis combine-les :

```
hit_intervals(shape, ray) → liste de paires (t_enter, t_exit, normal_enter, normal_exit)

Pour Union    : garde les intervalles qui sont dans A OU B
Pour Intersection : garde les intervalles qui sont dans A ET B
Pour Difference   : garde les intervalles de A NON couverts par B
```

Un objet convexe (sphère, box) donne au plus **un** intervalle `[t_enter, t_exit]`.

**Classe template :**

```cpp
enum class CsgOp { Union, Intersection, Difference };

template<CsgOp Op>
class CsgNode : public Shape {
    std::shared_ptr<Shape> left, right;

public:
    std::optional<HitRecord> hit(const Ray& r, double tmin, double tmax) const override;
};
```

**Indice sur les normales :** À l'entrée d'une intersection, la normale est celle de la surface touchée en dernier. À la sortie, c'est celle touchée en premier. Pour la différence A-B, les normales de B sont **inversées** (on est à l'intérieur de B).

**Fonctions factory :**
```cpp
auto csg_union(std::shared_ptr<Shape> a, std::shared_ptr<Shape> b);
auto csg_inter(std::shared_ptr<Shape> a, std::shared_ptr<Shape> b);
auto csg_diff (std::shared_ptr<Shape> a, std::shared_ptr<Shape> b);
```

## Exercice 5 — Tag dispatch pour la normale (30 min)

Le calcul de la normale dépend du type de primitive. Une façon propre sans `virtual` : **tag dispatch**.

```cpp
struct CylinderTag {};
struct ConeTag {};
struct BoxTag {};

// Surcharge pour chaque type de primitive
Vec3d compute_normal(const Vec3d& point, double radius, double height, CylinderTag);
Vec3d compute_normal(const Vec3d& point, double half_angle, double height, ConeTag);
Vec3d compute_normal(const Vec3d& point, const Vec3d& pmin, const Vec3d& pmax, BoxTag);
```

L'appelant détermine la surcharge via le tag, sans runtime check.

Utilise cette approche dans la classe `Cylinder` pour séparer proprement le calcul de normale du corps cylindrique vs les caps.
