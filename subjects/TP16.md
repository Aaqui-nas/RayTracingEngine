# TP16 — Textures et UV

## Objectif

Jusqu'ici les matériaux utilisent des couleurs solides. Ce TP introduit les **textures** :
des fonctions qui retournent une couleur à partir de coordonnées (u, v) sur la surface.

Ce TP introduit :
- Coordonnées UV dans `HitRecord`
- Interface `Texture` abstraite + `SolidColor`, `CheckerTexture`
- **UV sur Sphere** (mapping sphérique)
- **UV sur Triangle** (coordonnées par sommet + interpolation barycentrique)
- **ImageTexture** — charger un fichier PPM et l'appliquer comme texture
- **Bruit de Perlin** — texture procédurale lisse

---

## 1. Coordonnées UV

Les UV sont deux flottants dans [0, 1] qui repèrent un point sur la surface d'un objet.
Ils permettent de "coller" n'importe quelle texture 2D sur un objet 3D.

```
(u=0, v=0) ──────── (u=1, v=0)
     |                   |
     |   image texture   |
     |                   |
(u=0, v=1) ──────── (u=1, v=1)
```

---

## 2. Interface Texture

Crée `texture.h` avec la hiérarchie :

```cpp
class Texture {
public:
    virtual Vec3d sample(double u, double v, const Vec3d& point) const = 0;
    virtual ~Texture() {}
};

using TexturePtr = std::shared_ptr<Texture>;
```

Le paramètre `point` permet aux textures procédurales (Perlin, checker 3D) d'utiliser la position monde.

---

## 3. UV sur Sphere — mapping sphérique

Formule à partir de la normale sortante `n = (hit_point - center) / radius` :

```
theta = acos(-n.y)                      // angle depuis le pôle nord
phi   = atan2(-n.z, n.x) + pi          // angle autour de l'axe y

u = phi   / (2 * pi)                   // ∈ [0, 1]
v = theta / pi                         // ∈ [0, 1]
```

Points de repère :
| Normale | u | v |
|---------|---|---|
| (0, -1, 0) pôle sud | 0.5 | 0.0 |
| (0, +1, 0) pôle nord | 0.5 | 1.0 |
| (0, 0, +1) devant | 0.25 | 0.5 |
| (0, 0, -1) derrière | 0.75 | 0.5 |

## 4. UV sur Triangle — interpolation barycentrique

Chaque sommet porte ses propres UV. Le point d'impact est interpolé avec les coordonnées barycentriques `(bar_u, bar_v)` déjà calculées par Möller-Trumbore :

```
uv_hit = (1 - bar_u - bar_v) * uv_A
       +  bar_u               * uv_B
       +  bar_v               * uv_C
```

Le constructeur du Triangle reçoit trois paires UV supplémentaires :

```cpp
Triangle(Vec3d A, Vec3d B, Vec3d C,
         Vec3d uv_A, Vec3d uv_B, Vec3d uv_C,  // z ignoré, seuls x et y utilisés
         Material mat);
```

> Pour la compatibilité, garde l'ancien constructeur avec des UV par défaut : `(0,0,0)`, `(1,0,0)`, `(0,1,0)`.

## 5. ImageTexture — charger un fichier PPM

### Format PPM P3 (ASCII)

```
P3
<width> <height>
255
r g b  r g b  ...   (width*height triplets)
```

Charge dans un `std::vector<uint8_t>` de taille `width * height * 3`.

### Sampling

```cpp
int px = (int)(u * width)  % width;
int py = (int)(v * height) % height;
int idx = (py * width + px) * 3;
return Vec3d(data[idx], data[idx+1], data[idx+2]);
```

## 6. Bruit de Perlin

Le **bruit de Perlin** est une texture procédurale lisse.
Il repose sur une table de permutation de 256 entiers et des vecteurs gradient.

Structure suggérée :

```cpp
class PerlinNoise {
    static const int N = 256;
    int perm[N];          // table de permutation
    Vec3d grad[N];        // gradients unitaires aléatoires

public:
    PerlinNoise();        // initialise perm et grad avec un RNG fixe ou aléatoire
    double noise(const Vec3d& p) const;       // valeur ∈ [-1, 1]
    double turbulence(const Vec3d& p, int octaves = 7) const;  // ∈ [0, 1]
};
```

**`noise(p)`** : interpolation trilinéaire des gradients aux 8 coins du cube unité.

**`turbulence(p)`** : somme des valeurs absolues de noise à fréquences croissantes :
```
sum = 0
weight = 1
for i in range(octaves):
    sum    += weight * abs(noise(p))
    weight *= 0.5
    p      *= 2
return sum / (2 - 2^(1-octaves))   // normalise ≈ [0, 1]
```

## 8. Intégration avec les matériaux

Pour utiliser une texture dans un matériau lambertien, l'idée est de capturer un `TexturePtr` dans la lambda :

```cpp
auto lambertian_tex = [](TexturePtr tex) -> Material {
    return [tex](const Ray&, const HitRecord& rec) -> std::optional<Scatter> {
        Vec3d color = tex->sample(rec.u, rec.v, rec.point);
        Vec3d dir   = rec.normal + random_unit_vector();
        return Scatter{color, Ray(rec.point, dir)};
    };
};
```
