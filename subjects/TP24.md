## 1. Le modèle SIMD

### Registres et largeur

| Extension | Largeur | Doubles | Floats |
|-----------|---------|---------|--------|
| SSE2 (128-bit) | `__m128d` | 2 | 4 (`__m128`) |
| AVX (256-bit) | `__m256d` | 4 | 8 (`__m256`) |
| AVX-512 (512-bit) | `__m512d` | 8 | 16 |

Un `__m256d` contient 4 `double` dans un seul registre 256-bit. Une instruction `_mm256_add_pd` additionne les 4 paires simultanément — au lieu de 4 instructions ADD séparées.

### Cycle de vie d'un calcul SIMD

```
1. Charger des données du heap vers un registre    → _mm256_load_pd / _mm256_set_pd
2. Calculer sur le registre                        → _mm256_add_pd, _mm256_mul_pd, ...
3. Réduire ou stocker le résultat                  → _mm256_store_pd, réduction manuelle
```

### Alignement mémoire

Les instructions SIMD `_load` (sans `u`) exigent que les données soient alignées sur 32 octets. Si l'adresse n'est pas un multiple de 32, le programme plante.

```cpp
alignas(32) double data[4];   // garanti aligné sur 32 octets
```

## 2. Intrinsics utiles

Voici les intrinsics AVX que tu vas utiliser :

| Intrinsic | Opération |
|-----------|-----------|
| `_mm256_set_pd(d3,d2,d1,d0)` | Crée un `__m256d` avec 4 doubles *(ordre inversé !)* |
| `_mm256_load_pd(ptr)` | Charge 4 doubles depuis `ptr` aligné sur 32 |
| `_mm256_loadu_pd(ptr)` | Charge 4 doubles depuis `ptr` non-aligné (plus lent) |
| `_mm256_add_pd(a, b)` | Addition composante par composante |
| `_mm256_mul_pd(a, b)` | Multiplication composante par composante |
| `_mm256_hadd_pd(a, b)` | Réduction horizontale (addition de paires) |
| `_mm256_store_pd(ptr, a)` | Stocke 4 doubles vers `ptr` aligné |
| `_mm256_storeu_pd(ptr, a)` | Stocke 4 doubles vers `ptr` non-aligné |

Pour les inclure :
```cpp
#include <immintrin.h>   // AVX
```

Et pour activer AVX dans CMake :
```cmake
target_compile_options(raytracer PRIVATE -mavx2 -mfma)
```

## 3. std::chrono pour mesurer le temps

```cpp
#include <chrono>
using Clock = std::chrono::high_resolution_clock;

auto t0 = Clock::now();
// ... code à mesurer ...
auto t1 = Clock::now();
auto ms = std::chrono::duration_cast<std::chrono::microseconds>(t1 - t0).count();
```

Pour un benchmark fiable :
- Répète l'opération au moins `1'000'000` fois (évite la résolution basse du clock)
- Utilise `volatile` ou `[[maybe_unused]]` pour éviter que le compilateur supprime le code
- Mesure le **temps total / N itérations**, pas un seul appel

## Exercice 2 — Dot product AVX (4 vecteurs simultanés) (90 min)

Crée `src/core/simd.h`. L'idée : calculer `dot(a0,b0)`, `dot(a1,b1)`, `dot(a2,b2)`, `dot(a3,b3)` en **une seule séquence d'instructions AVX**.

**Structure de données — SoA (Structure of Arrays) :**

Au lieu de 4 vecteurs `{x0,y0,z0}`, `{x1,y1,z1}`, ..., arrange les composantes par dimension :

```cpp
struct Vec3x4 {
    alignas(32) double x[4];
    alignas(32) double y[4];
    alignas(32) double z[4];
};
```

**Algorithme du dot product AVX :**

```
1. Charge les 4 composantes x dans __m256d ax = _mm256_load_pd(a.x)
   Idem pour bx, ay, by, az, bz

2. Calcule xx = _mm256_mul_pd(ax, bx)
              yy = _mm256_mul_pd(ay, by)
              zz = _mm256_mul_pd(az, bz)

3. Somme :    res = xx + yy + zz  (addition composante par composante)

4. Stocke :   _mm256_store_pd(results, res)
```

`results` est un tableau de 4 doubles qui contient `{dot(a0,b0), dot(a1,b1), dot(a2,b2), dot(a3,b3)}`.

**Signature :**
```cpp
void dot4(const Vec3x4& a, const Vec3x4& b, double results[4]);
```

**Vérification manuelle :**
```
a0 = {1,2,3}, b0 = {1,0,0}  → dot = 1
a1 = {1,2,3}, b1 = {0,1,0}  → dot = 2
a2 = {1,2,3}, b2 = {0,0,1}  → dot = 3
a3 = {1,2,3}, b3 = {1,1,1}  → dot = 6
```

## Exercice 5 — Normalisation AVX (60 min)

Normaliser un vecteur = diviser par sa norme. Pour 4 vecteurs :

```
1. Calcule les 4 length_squared avec length_sq4
2. Prends la racine carrée : _mm256_sqrt_pd
3. Divise chaque composante : _mm256_div_pd
```

**Signature :**
```cpp
Vec3x4 normalize4(const Vec3x4& v);
```

**Astuce — reciprocal sqrt plus rapide :**

L'instruction `_mm256_rsqrt_ps` (floats seulement) donne `1/sqrt(x)`. Pour les doubles, utilise `_mm256_sqrt_pd` puis `1.0 / length`.

En pratique, pour les doubles, `_mm256_sqrt_pd` + `_mm256_div_pd` est souvent plus précis et suffisamment rapide.

## Exercice 6 — Ray4 : 4 rayons simultanés (120 min, extension)

Le vrai gain SIMD pour le ray tracer : shooter 4 rayons vers 4 pixels voisins et tester l'intersection avec une sphère en une seule passe.

**Structure :**
```cpp
struct Ray4 {
    Vec3x4 origin;
    Vec3x4 direction;
    // direction doit être normalisée
};
```

**Intersection Sphere/Ray4 :**

L'équation d'intersection scalaire est :
```
t² · d·d + 2t · (oc·d) + (oc·oc - r²) = 0
discriminant = (oc·d)² - (d·d)(oc·oc - r²)
```

Chaque produit scalaire peut être calculé avec `dot4`. Le discriminant (et sa racine carrée) se calcule avec des intrinsics AVX.

**Algorithme :**

```
1. oc = origin - center (soustraction AVX sur Vec3x4)
2. a = dot4(d, d)
3. half_b = dot4(oc, d)
4. c = dot4(oc, oc) - r*r  (soustraction d'un scalaire broadcasté)
5. discriminant = half_b*half_b - a*c  (tout en AVX)
6. test discriminant >= 0 avec _mm256_cmp_pd
7. pour les hits : t = (-half_b - sqrt(disc)) / a
```

**Indice pour broadcaster un scalaire :**
```cpp
__m256d radius2 = _mm256_set1_pd(r * r);  // {r², r², r², r²}
```

**Résultat attendu :** un masque AVX indiquant quels des 4 rayons ont intersecté la sphère, et les valeurs de t correspondantes.
