# TP15 — Caméra libre : lookAt, FOV, Profondeur de champ

## Objectif

La caméra actuelle est clouée à l'origine, regarde toujours vers `-z`. Pour explorer une scène, il faut pouvoir la placer n'importe où et la pointer vers n'importe quelle cible.

Ce TP introduit :
- **lookAt** : positionner la caméra avec `from` / `to` / `up`
- **FOV** : déjà partiellement là, à intégrer dans la nouvelle interface
- **Profondeur de champ** : les objets hors focus sont flous (aperture + focus distance)

---

## 1. Construire une base orthonormée (lookAt)

La caméra a besoin de 3 vecteurs perpendiculaires qui définissent son orientation dans l'espace :

```
         up
          |
          |
 ─────────+───────── u (droite)
         /|
        / |
       /  |
      w   v (haut recalculé)
 (arrière)
```

Donnés `from` (position), `to` (cible), `vup` (vertical global) :

```
w = normalize(from - to)         // axe "arrière" (opposé au regard)
u = normalize(cross(vup, w))     // axe "droite"
v = cross(w, u)                  // axe "haut" recalculé (orthogonal garanti)
```

> Pourquoi recalculer `v` ? Parce que `vup` n'est pas forcément perpendiculaire à `w`. Le cross product donne automatiquement la verticalité correcte.

Ensuite, les coins du plan image à distance `focus_dist` :

```
half_h = tan(vfov/2)
half_w = aspect * half_h

lower_left = from - u*(half_w*focus_dist) - v*(half_h*focus_dist) - w*focus_dist
horizontal = 2 * u * half_w * focus_dist
vertical   = 2 * v * half_h * focus_dist
```

## 2. Profondeur de champ

Un appareil photo réel a une **lentille** de diamètre non nul (l'ouverture = `aperture`).
Les rayons ne partent pas tous du même point — ils partent d'un **disque** autour de `from`.

```
lens_radius = aperture / 2

// Dans get_ray(s, t) :
rd     = lens_radius * random_in_unit_disk()
offset = u * rd.x + v * rd.y

origin    = from + offset
direction = lower_left + s*horizontal + t*vertical - from - offset
```

Avec `aperture = 0` → `offset = (0,0,0)` → comportement identique à avant.

**`random_in_unit_disk()`** : génère un point aléatoire dans le disque unité.
Méthode de rejection : générer (x, y) dans [-1, 1]² jusqu'à ce que x²+y²<1.

## 3. Chargement depuis scene.txt

Ajoute une ligne `camera` dans le format de scène :

```
camera  from_x from_y from_z   to_x to_y to_z   up_x up_y up_z   vfov   aperture   focus_dist
```

Exemple :
```
camera  0 1.5 4   0 0.5 0   0 1 0   60   0.05   4.0
```

## Exercices

### Étape 1 — Nouveau constructeur lookAt

Dans `camera.h`, ajoute un constructeur :

```cpp
Camera(Vec3d from, Vec3d to, Vec3d vup,
       double vfov_deg, double aspect_ratio,
       double aperture = 0.0, double focus_dist = 1.0);
```

Ce constructeur doit :
1. Calculer `w`, `u`, `v` (base orthonormée)
2. Calculer `half_h`, `half_w` à partir de `vfov_deg` et `aspect_ratio`
3. Calculer `lower_left_corner`, `horizontal`, `vertical` en tenant compte de `focus_dist`
4. Stocker `from`, `u`, `v`, `lens_radius = aperture / 2`

> Les champs `u`, `v` et `lens_radius` doivent être stockés car `get_ray()` en a besoin pour le DOF.

---

### Étape 2 — Modifier `get_ray()`

```cpp
Ray get_ray(double s, double t) const;
```

Sans DOF (`lens_radius == 0`) : comportement inchangé.
Avec DOF : l'origine est décalée par `offset = u * rd.x + v * rd.y`.

Pour le disque aléatoire, utilise le RNG déjà présent dans `materials.h` :
```cpp
#include "materials.h"  // rng, rng_dist
```
