# TP10 — Multithreading

## Objectif final
Paralléliser le rendu : chaque thread traite un bloc de lignes, réduisant le temps de calcul sur les machines multi-cœurs.

---

## 1. `std::thread`

```cpp
#include <thread>

void work(int id) {
    std::cout << "thread " << id << "\n";
}

std::thread t1(work, 1);
std::thread t2(work, 2);
t1.join();  // attend que t1 se termine
t2.join();
```

---

## 2. Race conditions et `std::mutex`

Si plusieurs threads écrivent au même endroit, les données peuvent être corrompues :

```cpp
#include <mutex>

std::mutex mtx;

void safe_write(int val) {
    std::lock_guard<std::mutex> lock(mtx);  // libéré automatiquement
    std::cout << val << "\n";
}
```

---

## 3. `std::atomic`

Pour des compteurs partagés simples, `std::atomic` est plus léger qu'un mutex :

```cpp
#include <atomic>

std::atomic<int> counter(0);
counter++;  // thread-safe
```

---

## 4. Stratégie pour le ray tracer

- Découpe l'image en bandes horizontales (une par thread)
- Chaque thread écrit dans sa portion du buffer `vector<Pixel>`
- Le thread principal écrit le fichier une fois tous les threads terminés

Attention : `std::cout` n'est pas thread-safe — écris dans un buffer d'abord (tu l'as fait au TP8).

2. Divise l'image en `N` bandes (N = nombre de cœurs)
3. Lance N threads, chacun rend sa bande dans `pixels[start..end]`
4. `join()` tous les threads
5. Écris le fichier PPM depuis le thread principal
6. Ajoute un compteur de lignes traitées avec `std::atomic<int>` et affiche la progression sur `std::cerr`
