# TP31 — GPU avec CUDA

**Durée estimée : ~20h**
**C++ introduit :** `__device__`, `__global__`, `__host__`, `cudaMallocManaged`, grille/bloc/thread, cuRAND
**Feature graphique :** Portage du path tracer sur GPU — gain ×100–1000 en performance

---

## Objectif

Le ray tracing est intrinsèquement parallèle : chaque pixel peut être calculé indépendamment. Jusqu'ici, on utilisait `N` threads CPU. Un GPU moderne a **des milliers** de cœurs — NVIDIA RTX 4090 : 16 384 CUDA cores.

Ce TP porte le path tracer sur GPU avec CUDA. C'est un saut important : le modèle de programmation est différent, les pièges sont nombreux, mais le gain est dramatique.

---

## 1. Modèle de programmation CUDA

### Hiérarchie d'exécution

```
Grid  (= lancement d'un kernel)
 └── Blocks (groupes de threads, jusqu'à 1024 threads/bloc)
       └── Threads (unité d'exécution)
```

Chaque thread connaît sa position via :
```cpp
int tid_x = blockIdx.x * blockDim.x + threadIdx.x;
int tid_y = blockIdx.y * blockDim.y + threadIdx.y;
```

Pour une image 800×600 avec des blocs de 16×16 :
```cpp
dim3 blocks_per_grid (800/16, 600/16);  // 50 × 37 blocs
dim3 threads_per_block(16, 16);          // 256 threads/bloc
render<<<blocks_per_grid, threads_per_block>>>(...)
```

### Qualificateurs de fonction

| Qualificateur | Appelé depuis | S'exécute sur |
|---|---|---|
| `__global__` | CPU (host) | GPU (device) |
| `__device__` | GPU seulement | GPU seulement |
| `__host__` | CPU (défaut) | CPU seulement |
| `__host__ __device__` | CPU ou GPU | selon l'appelant |

### Mémoire unifiée (Unified Memory)

```cpp
float* ptr;
cudaMallocManaged(&ptr, size);   // accessible CPU et GPU
// ... utilise ptr sur CPU et GPU ...
cudaFree(ptr);
```

La mémoire unifiée simplifie le code (pas de copie explicite), mais avec un coût en performance. Pour un renderer, c'est le bon compromis pour démarrer.

---

## 2. Contraintes du code CUDA

Le GPU **ne peut pas** :
- Appeler `new` / `delete` arbitrairement dans un kernel (depuis CUDA 4+, c'est possible mais coûteux — à éviter)
- Utiliser `std::vector`, `std::shared_ptr`, `std::function`
- Lancer des exceptions
- Accéder à la mémoire CPU depuis le GPU (sauf unified memory)

Il faut reécrire les structures de données en plain arrays.

## Exercice 1 — "Hello GPU" : premier kernel (30 min)

Crée `src/gpu/hello_cuda.cu`. Par convention, les fichiers CUDA ont l'extension `.cu`.

```cpp
#include <cstdio>

__global__ void hello_kernel(int n) {
    int tid = blockIdx.x * blockDim.x + threadIdx.x;
    if (tid < n)
        printf("Hello from thread %d\n", tid);
}

int main() {
    hello_kernel<<<1, 32>>>(20);
    cudaDeviceSynchronize();   // attend que le kernel termine
    return 0;
}
```

## Exercice 2 — Vec3 et Ray sur GPU (45 min)

Le `Vec3d` actuel utilise `double` avec des méthodes inline — il peut fonctionner sur GPU avec des modifications mineures.

Crée `src/gpu/cuda_vec3.h` :

```cpp
struct CudaVec3 {
    float x, y, z;   // float sur GPU (plus rapide que double sur la plupart des GPUs)

    __host__ __device__ CudaVec3(float x, float y, float z) : x(x), y(y), z(z) {}
    __host__ __device__ CudaVec3 operator+(const CudaVec3& b) const { return {x+b.x, y+b.y, z+b.z}; }
    __host__ __device__ CudaVec3 operator*(float t) const { return {x*t, y*t, z*t}; }
    __host__ __device__ float dot(const CudaVec3& b) const { return x*b.x + y*b.y + z*b.z; }
    __host__ __device__ CudaVec3 normalize() const {
        float len = sqrtf(x*x + y*y + z*z);
        return {x/len, y/len, z/len};
    }
    // ... autres opérateurs
};

struct CudaRay {
    CudaVec3 origin, direction;
    __host__ __device__ CudaVec3 at(float t) const { return origin + direction * t; }
};
```

**Règle d'or :** tout ce qui sera appelé depuis un kernel doit être `__device__` ou `__host__ __device__`.

## Exercice 4 — Scène GPU et kernel de rendu (90 min)

La scène GPU est un tableau plat de sphères (pas de `shared_ptr`, pas de `vector`) :

```cpp
struct CudaScene {
    CudaSphere* spheres;   // pointeur unified memory
    int num_spheres;
};

struct CudaMaterial {
    enum Type { LAMBERTIAN, METAL, DIELECTRIC };
    Type type;
    CudaVec3 albedo;
    float roughness;
    float ior;
};
```

**Kernel de rendu :**

```cpp
__global__ void render_kernel(
    CudaVec3* framebuffer,   // buffer de sortie, un CudaVec3 par pixel
    int width, int height,
    CudaScene scene,
    CudaMaterial* materials,
    int samples_per_pixel,
    CudaCamera camera,
    curandState* rand_states)   // un état random par thread
{
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    if (x >= width || y >= height) return;

    int pixel_idx = y * width + x;
    curandState local_rand = rand_states[pixel_idx];

    CudaVec3 color(0, 0, 0);
    for (int s = 0; s < samples_per_pixel; s++) {
        float u = (x + curand_uniform(&local_rand)) / float(width);
        float v = (y + curand_uniform(&local_rand)) / float(height);
        CudaRay ray = camera.get_ray(u, v, &local_rand);
        color = color + ray_color(ray, scene, materials, &local_rand, 50);
    }
    color = color * (1.0f / samples_per_pixel);

    rand_states[pixel_idx] = local_rand;
    framebuffer[pixel_idx] = color;
}
```

**Initialisation des états random (kernel séparé) :**

```cpp
__global__ void init_rand_states(curandState* states, int width, int height, unsigned int seed) {
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    if (x >= width || y >= height) return;
    int idx = y * width + x;
    curand_init(seed, idx, 0, &states[idx]);
}
```

## Exercice 5 — ray_color récursif sur GPU (60 min)

Les fonctions récursives sur GPU **sont possibles** mais limitées (profondeur de pile fixe, ~500 appels max).

Préfère une version **itérative** (plus sûre sur GPU) :

```cpp
__device__ CudaVec3 ray_color(
    CudaRay ray,
    const CudaScene& scene,
    const CudaMaterial* materials,
    curandState* rng,
    int max_depth)
{
    CudaVec3 throughput(1, 1, 1);
    CudaVec3 color(0, 0, 0);

    for (int depth = 0; depth < max_depth; depth++) {
        CudaHitRecord rec;
        if (!closest_hit(scene, ray, 0.001f, 1e30f, rec)) {
            // Background
            float t = 0.5f * (ray.direction.normalize().y + 1.0f);
            CudaVec3 bg = CudaVec3(1,1,1)*(1-t) + CudaVec3(0.5f,0.7f,1.0f)*t;
            color = color + throughput * bg;
            break;
        }

        // Applique le matériau
        const CudaMaterial& mat = materials[rec.material_idx];
        CudaVec3 attenuation;
        CudaRay scattered;
        if (scatter_material(mat, ray, rec, attenuation, scattered, rng)) {
            throughput = throughput * attenuation;
            ray = scattered;
        } else {
            break;
        }
    }
    return color;
}
```

## Exercice 6 — Pipeline complet et comparaison (45 min)

Assemble le renderer GPU complet :

```cpp
int main() {
    const int W = 800, H = 600, SAMPLES = 128;

    // 1. Alloue framebuffer en unified memory
    CudaVec3* fb;
    cudaMallocManaged(&fb, W * H * sizeof(CudaVec3));

    // 2. Alloue et initialise la scène
    CudaScene scene = create_scene();   // alloue en unified memory

    // 3. Init états random
    curandState* rand_states;
    cudaMallocManaged(&rand_states, W * H * sizeof(curandState));
    dim3 blocks(W/16+1, H/16+1), threads(16, 16);
    init_rand_states<<<blocks, threads>>>(rand_states, W, H, 42);
    cudaDeviceSynchronize();

    // 4. Rendu
    auto t0 = std::chrono::high_resolution_clock::now();
    render_kernel<<<blocks, threads>>>(fb, W, H, scene, /*...*/, SAMPLES, camera, rand_states);
    cudaDeviceSynchronize();
    auto t1 = std::chrono::high_resolution_clock::now();

    // 5. Écriture PPM
    write_ppm("output_gpu.ppm", fb, W, H);
    cudaFree(fb);
}
```

**Compare :**
- CPU avec 8 threads (TP28) : X ms
- GPU avec `blocks = (W/16, H/16)` : Y ms
- Speedup = X / Y

## Points d'attention et pièges fréquents

**Erreur 1 : utiliser `std::function` dans un kernel**  
`std::function` fait des allocations → incompatible. Remplace par un `switch` sur un enum de matériau.

**Erreur 2 : oublier `cudaDeviceSynchronize()`**  
Le lancement de kernel est asynchrone. Sans synchronisation, le programme se termine avant la fin.

**Erreur 3 : accéder à la mémoire CPU depuis le kernel**  
Toute donnée lue dans le kernel doit être en device memory ou unified memory.

**Erreur 4 : stack overflow**  
La pile d'un thread CUDA est petite (~1KB). Évite les grandes variables locales et la récursion profonde.

- [ ] `hello_kernel` s'affiche correctement
